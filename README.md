# Recipe Storage (レシピ保管・整理Webアプリケーション)

家庭で使える料理レシピを保管して整理するWebアプリケーションです。手動での登録に加え、レシピブログなどのプレーンテキストを貼り付けるだけで、AI（Gemini API）が料理名、人数、材料、手順を自動認識してフォームに入力する「AIインポート機能」を搭載しています。

---

## 🛠️ ローカル開発環境の起動手順

本アプリケーションをローカル環境で動かすための手順です。

### 1. 前提条件
- **Node.js** (v18以上推奨)
- **Docker** & **Docker Compose** がインストールされ、起動していること

### 2. データベースの起動 (Docker)
ローカルの PostgreSQL データベースを Docker コンテナとして起動します。
プロジェクトのルートディレクトリで以下のコマンドを実行してください。

```bash
docker-compose up -d
```
*※これで、バックグラウンドで PostgreSQL コンテナ（ポート `5432`）が起動します。*

### 3. 環境変数の設定
プロジェクトのルートにある `.env.local` ファイルを作成（または編集）し、以下の内容を設定します。

```env
DATABASE_URL="postgresql://postgres:postgrespassword@localhost:5432/recipe_storage?schema=public"

# AIインポート機能を利用するためのGemini APIキーを設定してください
GEMINI_API_KEY="ここにあなたのGemini_APIキーを入力"
```

### 4. データベースの初期設定 (マイグレーション & シード)
テーブルの作成と、初期サンプルデータ（カテゴリやサンプルレシピ）の投入を行います。

```bash
# マイグレーションの実行（テーブル作成）
npx prisma migrate dev --name init

# 初期サンプルデータの投入（シード実行）
npx prisma db seed
```

### 5. 開発サーバーの起動
必要なパッケージをインストールし、Next.jsの開発サーバーを起動します。

```bash
# パッケージのインストール
npm install

# 開発サーバーの起動
npm run dev
```

起動後、ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスしてください。

## ⚡ 主な機能
- **3カラムダッシュボード UI**: PCの大画面を活かし、お買い物リストを常時右側に配置したシームレスな操作性。
- **AIインポート**: 雑多なレシピテキストをコピペするだけで、材料（分量・単位を分離）や手順をAIが自動解析して取り込みます。
- **動的サービングスケール**: 詳細画面で人数（例: 2人前 -> 4人前）を変更すると、材料の分量が自動計算されます。
- **お買い物リスト統合**: レシピ詳細から必要な材料をチェックして、ワンクリックでお買い物リストに一括追加できます。
