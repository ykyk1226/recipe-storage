# 料理レシピ保管・整理Webアプリケーション（Recipe Storage）仕様書

本書は、家庭向けの料理レシピを保管・整理するWebアプリケーションの開発用仕様書です。
開発を担当するAIやエンジニアが、手戻りなく高品質な実装を行えるように技術構成、データモデル、機能要件、画面レイアウトを定義します。

---

## 1. プロジェクト概要

*   **アプリ名**: Recipe Storage (レシピ・ストレージ)
*   **目的**: 家庭内で日常的に使うレシピを1か所にまとめ、効率的に保管・整理・検索できるWebアプリケーション。
*   **ターゲット**: 自炊をする人、家族の献立を管理する人。
*   **提供価値**: 
    *   Web上のレシピや自己流のレシピをすっきり整理できる。
    *   冷蔵庫にある材料からレシピを素早く見つけられる。
    *   作りたいレシピからお買い物リストを自動で生成し、買い忘れを防ぐ。
    *   **AIインポート**: ブログやSNS、メモなどの雑多なテキストから、料理名・材料・手順をAIが自動認識して一瞬でレシピ登録できる。

---

## 2. 技術スタック

*   **フロントエンド**: 
    *   **フレームワーク**: Next.js 16 (App Router)
    *   **ライブラリ**: React 19
    *   **言語**: TypeScript
    *   **スタイリング**: TailwindCSS v4 (Vanilla CSSによる微調整)
    *   **アイコン**: Lucide React
*   **バックエンド / データベース**:
    *   **データベース**: PostgreSQL
    *   **ORM (Object-Relational Mapping)**: Prisma または Drizzle ORM
*   **AI / 自然言語処理 (レシピ自動解析)**:
    *   **API/SDK**: Vercel AI SDK または Gemini API (`gemini-2.5-flash` などの構造化出力機能 `responseSchema` を利用)
*   **ホスティング**: Vercel (フロントエンド/API) + PostgreSQLホスティング（Supabase, Neon, もしくはローカルDocker等）

## 3. データモデル設計 (Database Schema)

Prisma Schema形式でデータモデルを定義します。リレーションを適切に設定し、お気に入り、カテゴリ、お買い物リスト、材料フィルターに対応します。

```prisma
// This is a conceptual Prisma schema for the database configuration

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

// レシピモデル
model Recipe {
  id          String   @id @default(uuid())
  title       String   // レシピ名
  description String?  // レシピの説明・メモ
  servings    Int      @default(2) // 何人前か
  prepTime    Int?     // 下ごしらえ時間 (分)
  cookTime    Int?     // 調理時間 (分)
  image       String?  // レシピ画像のURL
  isFavorite  Boolean  @default(false) // お気に入りフラグ
  sourceUrl   String?  // 参考にしたWebサイトのURL
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  // リレーション
  ingredients Ingredient[]
  steps       Step[]
  categories  Category[] @relation("RecipeToCategory")
  shoppingItems ShoppingItem[]
}

// 材料モデル (材料フィルター用)
model Ingredient {
  id       String @id @default(uuid())
  name     String // 材料名 (例: "豚バラ肉", "キャベツ")
  amount   String // 分量 (例: "200", "1/4")
  unit     String // 単位 (例: "g", "個", "大さじ1")
  recipeId String
  recipe   Recipe @relation(fields: [recipeId], references: [id], onDelete: Cascade)

  @@index([name]) // 材料名での検索を高速化
}

// 作り方の手順モデル
model Step {
  id          String  @id @default(uuid())
  stepNumber  Int     // 手順番号 (1, 2, 3...)
  instruction String  // 手順の説明文
  image       String? // 手順ごとの画像URL
  recipeId    String
  recipe      Recipe  @relation(fields: [recipeId], references: [id], onDelete: Cascade)
}

// カテゴリモデル
model Category {
  id      String   @id @default(uuid())
  name    String   @unique // カテゴリ名 (例: "主菜", "副菜", "パスタ", "スープ")
  color   String   @default("#94a3b8") // カテゴリのテーマカラー (HEX)
  recipes Recipe[] @relation("RecipeToCategory")
}

// お買い物リストモデル
model ShoppingItem {
  id        String   @id @default(uuid())
  name      String   // 買うもの (材料名など)
  amount    String?  // 分量
  unit      String?  // 単位
  isChecked Boolean  @default(false) // 買い忘れ防止のチェック
  recipeId  String?  // どのレシピから追加されたか (任意)
  recipe    Recipe?  @relation(fields: [recipeId], references: [id], onDelete: SetNull)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

## 4. 機能要件

### 4.1 レシピ管理機能
*   **レシピ登録・編集**:
    *   レシピの基本情報（タイトル、説明、人数、所要時間、参考URL、画像）を入力可能。
    *   材料リスト（名称、分量、単位）を動的に追加・削除可能。
    *   調理手順（ステップ番号、説明、画像）を動的に追加・並べ替え可能。
*   **レシピ削除**:
    *   不要なレシピを削除可能。紐づく材料や手順もカスケード削除される。
*   **レシピ詳細表示**:
    *   登録されたレシピを綺麗にフォーマットして表示。
    *   「人数（サービングサイズ）」を変更した際に、材料の分量を自動で倍数計算する機能（動的スケール）。

### 4.2 レシピ検索・フィルタリング機能
*   **キーワード検索**:
    *   レシピ名、説明、材料名から部分一致でレシピを検索する。
*   **カテゴリフィルター**:
    *   「主菜」「副菜」「デザート」などのカテゴリを選択してレシピを絞り込む。複数選択も可能。
*   **お気に入りフィルター**:
    *   お気に入り（`isFavorite: true`）のレシピのみを表示するトグルスイッチ。
*   **材料フィルター**:
    *   「冷蔵庫にある材料」を入力し、その材料を含むレシピのみを絞り込む。
    *   （例: 「キャベツ」「豚肉」を入力すると、その両方、もしくはいえないかを含むレシピがヒットする）

### 4.3 お買い物リスト機能
*   **レシピからの自動追加**:
    *   レシピ詳細画面から「材料をお買い物リストに追加」ボタンを押すと、すべての材料が自動的にお買い物リストに登録される。
*   **手動追加・編集**:
    *   レシピ以外の買い物（調味料や日用品など）も自由に追加可能。
*   **チェックとクリア**:
    *   購入したアイテムはチェックを入れてグレーアウトできる。
    *   「チェック済みを削除」または「リストを全クリア」する機能。

### 4.4 テキストからのレシピ自動解析機能（AIインポート）
*   **概要**:
    *   Web上のテキスト、チャットのコピー、手書きメモなどの「構造化されていないプレーンテキスト」から、AIを用いて自動でレシピ構造（タイトル、材料、手順等）を解析・抽出する。
*   **動作詳細 / 処理フロー**:
    1.  **入力**: ユーザーがテキストボックスに、レシピが書かれた任意のテキストを入力または貼り付け（コピペ）する。
    2.  **解析**: 「AIで解析」ボタンを押すと、Next.jsのAPI Route（`/api/recipes/parse`）を経由してLLMにプロンプトを送信。
        *   **プロンプト仕様**: 入力テキストから `title` (料理名)、`servings` (人数)、`ingredients` (材料名、分量、単位のリスト)、`steps` (手順番号、説明のリスト) を抽出し、厳密なJSON形式で返却するようAIに指示（Geminiの `Structured Outputs` を利用し、データ型を保証）。
    3.  **プレビュー（重要）**: 解析が完了すると、自動的に「新規レシピ登録」の入力フォームに値が自動挿入される。
        *   ユーザーはフォーム上で、誤って解析された分量や表記の揺れ（例: 「1/2丁」と「半丁」など）を確認・修正できる。
    4.  **保存**: ユーザーが「保存する」ボタンを押すことで、PostgreSQLデータベースに正式登録される（確認なしで直接DB保存せず、確認ステップを挟むことでデータの品質を保つ）。

## 5. 画面設計・UI要件 (PC向け / デスクトップ専用)

PCの大画面を最大限に活用した、3カラム構成または分割画面のモダンなレイアウトを採用します。

### 5.1 全体レイアウト構成 (モダン・ダッシュボード型)
*   **左サイドバー (幅: 約260px)**
    *   アプリロゴ
    *   メインナビゲーション:
        *   「レシピ一覧」
        *   「新規レシピ登録」（クリック時に「通常登録」か「AIインポート」を選択可能）
        *   「カテゴリ管理」
    *   クイックフィルター:
        *   お気に入りトグル (星マーク)
        *   カテゴリ一覧（チェックボックス付き）
        *   材料フィルター（タグ入力形式で材料を追加して絞り込む）
*   **中央メインコンテンツ (幅: フレキシブル)**
    *   上部: 検索バー（リアルタイム検索）と並び替えオプション（作成日順、調理時間順）。
    *   グリッドエリア: レシピカード一覧。
        *   カードには、画像、タイトル、調理時間、お気に入りボタン、カテゴリタグを表示。
        *   カードホバー時に浮かび上がるようなソフトな影と、わずかな拡大アニメーションを付与。
*   **右サイドバー (幅: 約320px - 固定または開閉式)**
    *   「お買い物リスト（Shopping List）」を常時表示。
    *   中央でレシピを見ながら、ワンクリックで右側のリストに材料を放り込める。
    *   チェックボックス、手動追加インプット、クリアボタンを配置。

### 5.2 主要画面の設計

#### 1. レシピ一覧（メインダッシュボード）
*   **特徴**: PC向けに最適化されたグリッドレイアウト（3〜4列）。
*   **デザイン**: 白・グレーを基調としたクリーンな背景に、レシピ写真が映えるように設計。
*   **材料フィルター**:
    *   「冷蔵庫の食材で探す」という入力エリアを左サイドバーに設置。
    *   ユーザーが入力してEnterを押すと、タグ（例: `[ 豚肉 x ]`, `[ キャベツ x ]`）が生成され、それらを元にレシピ一覧が即座に絞り込まれる。

#### 2. レシピ詳細・編集画面
*   **特徴**: 一覧からレシピカードをクリックすると、画面遷移せずに右側からスライドインする「詳細ドロワー」または「2分割スプリットビュー」を採用。
*   **レイアウト**:
    *   左半分: レシピのメイン画像、タイトル、調理時間、お買い物リストへの一括追加ボタン。
    *   右半分: スクロール可能な「材料リスト」と「手順（ステップ1, 2, 3...）」。
    *   「編集する」ボタンを押すと、その場でインライン、あるいは編集モーダルに切り替わる。

#### 3. 新規レシピ登録・編集画面（AIインポート連携）
*   **特徴**: 通常の手動入力フォームに加え、「AIテキストインポート」用のタブまたは切り替えUIを用意。
*   **AIインポート画面レイアウト**:
    *   **入力ビュー**: 画面の左半分に大きなテキストエリアを配置（「ここにレシピのテキストを貼り付けてください」というプレースホルダー）。右半分には「解析結果プレビュー」プレースホルダー。
    *   **解析実行**: テキスト入力後に「AIで材料・手順を解析する」ボタン（キラキラしたエフェクトやローディングアニメーション付きのモダンなボタン）を押す。
    *   **編集ビュー（解析後）**: 解析が完了すると、左側のテキストエリアが縮小し、右側のフォームエリアに「解析されたタイトル」「自動生成された材料リスト（行追加UI）」「自動生成された手順リスト」が挿入される。
    *   ユーザーはその場で最終調整を行い、「レシピを登録する」ボタンをクリックして保存する。

## 6. デザインシステム & ビジュアルスタイル (モダンデザイン)

*   **配色 (Color Palette)**
    *   **メインテーマ**: 温かみがありつつも洗練された「モダン・オーガニック」
    *   **背景色**: オフホワイト（`#f8fafc` - Slate 50）
    *   **カード・コンポーネント背景**: 純白（`#ffffff`）に極薄いグレーのボーダー（`#e2e8f0` - Slate 200）
    *   **プライマリカラー**: 深みのあるフォレストグリーン（`#15803d` - Green 700）または暖かみのあるテラコッタ（`#c2410c` - Orange 700）
    *   **AI機能用のテーマカラー**: インディゴまたはバイオレット（`#6366f1` - Indigo 500 / AI解析ボタンやローディング効果に使用し、特別感を演出）
    *   **アクセント（お気に入りなど）**: ゴールド（`#eab308` - Yellow 500）
*   **タイポグラフィ**
    *   Google Fonts から `Inter` または `Outfit` を読み込み、モダンで視認性の高いフォント構成とする。
*   **インタラクション (マイクロアニメーション)**
    *   AI解析中の滑らかな骨組みローディング（Skeleton Screen）や、解析完了時にフォームにデータがフェードインするアニメーション。
    *   お買い物リストのアイテムをチェックした際、横線が引かれてフェードアウトするアニメーション。

## 7. 非機能要件 & 開発の注意点

*   **AI APIキーの管理**:
    *   Gemini API や OpenAI API のAPIキーは、クライアントサイドに露出させず、必ずNext.jsのサーバー環境変数（`.env.local`）に隠蔽し、API Routesを経由して呼び出すこと。
*   **プロンプト・エンジニアリング**:
    *   AIに渡すシステムプロンプトには、出力フォーマットをJSONに限定すること、日本語を正しく処理すること、想定外の入力に対してもエラーではなく空のレシピ構造を返すようロバストに設計すること。
*   **パフォーマンス**:
    *   AI解析は数秒かかるため、UX向上のために適切な進捗バーやローディング表示を行うこと。

*   **モバイル対応**: レスポンシブデザインの導入（スマートフォン向け表示）。
*   **画像からのレシピインポート (OCR & マルチモーダル)**: レシピ本の写真やスクリーンショットをアップロードするだけで、AI（マルチモーダルモデル）が画像内のテキストを読み取り、レシピとして自動解析・登録する機能。
